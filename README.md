# Worship Signal

Модульное веб-приложение для богослужения BFUPC. Без установки, статические файлы, GitHub Pages.

## Структура (блочная — каждый файл меняется независимо)

```
index.html              — точка входа, порядок загрузки скриптов
core/
  theme.css             — ВСЯ палитра и стили (меняй внешний вид только тут)
  store.js              — пространство имён WS, конфиг, localStorage
  auth.js               — пароли, уровни, идентичность устройства
  data.js               — чтение/запись JSON через GitHub API + кэш
  sync.js               — связь между устройствами (ntfy.sh, SSE)
  ui.js                 — общие помощники (el, тосты, модалки, авто-fit)
  app.js                — запуск и маршрутизация (загружается последним)
modules/
  pin.js                — экран входа
  role.js               — выбор роли
  settings.js           — настройки (имя, токен, выход, админка-заглушка)
  hymns.js              — Псалмы/Песни: списки + редактор
  stage.js              — Сцена: 4 квадранта
  chair.js              — Кафедра: Медиа/Текст/Чат/Библия
  operator.js           — Оператор: обычный режим + трансляция
  chat.js               — общий чат
data/
  songs.json            — песни
  psalms.json           — псалмы (псалмоспів)
  texts.json            — заготовки текстов
  announcements.json    — объявления (живут 2 дня)
  bible.json            — 66 книг (EN+UA, ссылки)
  media.json            — медиа (ссылки Google Drive)
```

## Пароли (уровни)

| Пароль          | Уровень  | Права                                         |
|-----------------|----------|-----------------------------------------------|
| `2580`          | Обычный  | смотреть, добавлять                           |
| `OPERATOR`      | Сцена    | + редактировать/удалять, роли Сцена/Псалмы    |
| `LIMEinCHURCH`  | Lime     | + назначение уровней, блокировка, бэкапы      |

Пароль вводится один раз, уровень сохраняется на устройстве. Пароль `OPERATOR` для
ролей Сцена/Псалмы действует 24 часа.

## Деплой

1. Создать репозиторий (рекомендую новый, напр. `worship-signal`), залить все файлы.
2. Settings → Pages → Branch `main` / root → Save.
3. Открыть `https://<логин>.github.io/worship-signal/`.

Репозиторий определяется автоматически из адреса. Менять ничего не нужно.

## Запись песен/псалмов (нужен токен)

Чтение работает сразу. Для сохранения изменений в базу:
1. GitHub → Settings → Developer settings → Personal access tokens → Fine-grained.
2. Доступ к нужному репозиторию, права **Contents: Read and write**.
3. В приложении: Настройки → вставить токен. Токен хранится только на устройстве.

## Канал связи

Все устройства общаются через общий ntfy-канал `worship-signal-bfupc-7c3a9`
(меняется в `core/store.js` → `TOPIC`). Бесплатно, без регистрации.

## Что готово

- Вход, уровни доступа, выбор роли, имя устройства, настройки.
- Псалмы/Песни: вкладки, список, просмотр блоков, отправка на проектор, редактор
  (куплет/припев/бридж, перевод, чекбокс показа перевода), запись в GitHub.
- Оператор: обычный режим (лента активности + чат + очистка проектора) и трансляция
  (чёрный экран, авто-подгонка текста, всегда горизонтально, контроль связи).
- Сцена: 4 квадранта + переход в Псалмы.
- Кафедра: Текст → проектор, Библия → оператору (EN+UA), Медиа (Drive embed), Чат.
- Чат с авто-очисткой через сутки.
- Lime-админка: онлайн-список устройств, смена уровня, блокировка на 3 мин (presence через ntfy, без сервера).
- Кафедра: Текст→проектор (с редактором), Объявления (с истечением 2 дня), Библия→оператору (EN+UA), Медиа (Drive embed), Чат.
- Программы служения: составление сценария (псалмы/песни/тексты/объявления/заметки), ведение по программе с отправкой блоков.
- Надёжная доставка сообщений: SSE + добор пропущенного (since при reconnect, страховочный poll).
- Надёжный проектор: контент рисуется через единый WS.Projector ВСЕГДА, независимо от экрана (чинит «то рисует, то нет»).
- Общий лог оператора: хранится в localStorage, виден любому оператору, переживает перезаход; кнопка «Очистить лог» (синхронно у всех).
- Медиа-менеджер Google Drive: добавление/правка ссылок на файлы, показ на проекторе.
- Технические настройки (GitHub-токен, репозиторий) доступны только Lime.

## Что заложено на потом (точки расширения готовы)

- Google Drive OAuth и еженедельные бэкапы.
- Тексты стихов Библии (поле зарезервировано, сейчас только ссылки).
- PWA (офлайн-кэш, экран не гаснет, иконка на телефон).

## PWA (устанавливается как приложение)
- manifest.json + service-worker.js + icons/ — офлайн-кэш всего приложения, иконка на домашний экран.
- Статика — cache-first; data/*.json — network-first (свежие, офлайн из кэша); ntfy/github/google — только сеть.
- Авто-обновление: новый service worker применяется сам (controllerchange → reload).
- Wake Lock: в режиме «Трансляция» экран не гаснет.

## Языки (UA / EN)
- core/i18n.js — два словаря (uk по умолчанию, en). WS.t('key', ...args) с подстановкой {0}.
- Весь интерфейс всех ролей локализован. Переключатель языка — Настройки (доступен всем ролям).
- Русский в интерфейсе заменён на украинский + английский.

## Надёжность проектора и логов (v2)
- Проектор и лог теперь строятся ИЗ общего потока ntfy — одинаково на всех устройствах.
- НЕ фильтруем «свои» сообщения (иначе логи расходились между устройствами).
- restore(): при старте добираем историю за 6ч — восстанавливаем последний слайд и лог даже при позднем подключении.
- isLive по времени последней активности (не readyState) — убрано ложное «нет связи» и мигание проектора.

## Режим ведения программы (презентер)
- Тап по пункту программы открывает предпросмотр.
- Стрелки ◀ ▶ — переход между пунктами служения.
- Внизу 2 кнопки: [⚑ Оператору] (запрос оператору — все уровни) и [На экран] (показ, Сцена+).
- Песня/псалом — блоки с тап-отправкой; текст/объявление — вывод на экран. Очистка экрана — в шапке (Сцена+).

## Google Drive: загрузка файлов + резервные копии (v3)
- core/drive.js — OAuth через Google Identity Services (область drive.file), прямые вызовы Drive REST.
- Настройки (Lime): Google Client ID, «Подключить», «Резервная копия сейчас», «Восстановить из последней копии», дата последней копии + напоминание раз в неделю.
- Кафедра → Медиа → «Загрузить с устройства»: файл грузится на Drive, делается публичным, добавляется в media.json и сразу доступен на проекторе.
- Бэкап: все коллекции (кроме Библии) в один JSON worship-signal-backup-ДАТА.json на Drive. Восстановление перезаписывает коллекции в GitHub (нужен GitHub-токен Lime).
- Требуется настройка Google Cloud (Client ID) — см. инструкцию.
